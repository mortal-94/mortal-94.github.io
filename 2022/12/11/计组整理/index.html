计组整理

我认为计算机组成原理大致分为下面这几部分：

一些电路知识、基本组成（概论）、信息编码与数据表示、运算方法和运算器构成、存储系统、指令系统、控制器的操作、输入输出系统、总线。

PS：本人没学过数电，电路知识就不在这整理了。

概论

计算机系统

计算机系统有两个部分： 硬件系统、软件系统 。

硬件系统

计算机硬件系统是由 存储器、运算器、控制器、输入设备、输出设备 组成。

模块 功能
存储器 用于存放指令和数据
运算器 执行加减乘除运算、逻辑运算
控制器 译码分析指令，产生并发生控制信号以保证计算机协调地工作；整个计算机的控制中心
输入设备 计算机输入信息的设备
输出设备 将计算机的处理结果转换成人或其他设备可以接收的形式。

PS：由于运算器和控制器在逻辑关系和电路结构上的联系十分紧密，通常将它们制作在同一芯片上，称之为 中央处理器（Central Processing Unit，CPU） 。

软件系统

计算机软件系统可以分为两大类： 系统软件、应用软件 。

模块 功能
系统软件 管理、调度、监视、和维护系统资源的程序；这些系统资源包括硬件资源和软件资源；使系统得到合理的调度，确保高效运行
应用软件 用户根据具体任务需求所开发的程序

常用的系统软件有： 操作系统、语言处理程序、数据库管理系统、服务型程序、网络管理软件 。 应用软件有很多。

系统软件 功能
操作系统 协调管理资源、组织运行、提供计算机和用户之间的交互接口；最基本、核心的系统软件。
语言处理程序 翻译高级语言编写的程序供计算机执行
数据库管理系统 管理数据库
服务型程序 帮助用户使用和维护计算机，提供服务性手段的程序
网络管理软件 主要指网络通信协议和网络操作系统。

层次结构

计算机系统的层次结构是指将计算机系统抽象为不同层次的模型，以便了解和研究计算机系统的运作方式。

层次 作用
微程序设计级 完成计算机系统的基本运算和存储功能，包括处理数据、存储数据和进行输入/输出操作。
机器指令系统级 提供计算机硬件可以读懂并可直接操纵其工作的二进制信息。软硬分界。
操作系统级 管理硬件资源、支持其他软件，使计算机自动、高效地运行
语言处理程序及其他系统软件 翻译高级语言编写的程序供计算机执行、使用户更好地使用计算机
应用程序 完成用户的具体任务

计算机的性能指标

主要讨论计算机硬件的技术指标。

有如下几个方面： 机器字长、存储容量、运算速度、可配置的外围设备、性能价格比、可靠性可维护性及可用性 。

指标 描述
机器字长 CPU一次能处理的数据位数。决定了寄存器、运算部件、数据总线的位数。
存储容量 主存储器和辅助存储器的容量。
运算速度 目前普遍采用单位时间执行指令的平均条数来衡量。MIPS (Million Instruction Per Second)
可配置的外围设备
性能价格比
可靠性、可维护性及可用性 MTBF(Mean Time Between Failure)，MTRF(Mean Time to Repair Fault)，MTBF/(MTBF+MTRF)

什么是机器字长？它对计算机性能有哪些影响？

指 CPU 一次能处理的数据位数。它影响着计算机的运算速度，硬件成本、指令系统功能，数据处理精度等。

什么是计算机软、硬件的逻辑等价性？理解软硬件的等价性的意义是什么？

计算机软件和硬件的逻辑等价性指的是在实现相同的逻辑功能时，软件和硬件的行为是相同的。这意味着，如果一个特定的逻辑功能可以用软件实现，那么它也可以用硬件来实现，反之亦然。

理解软硬件的等价性的意义在于，它有助于我们在设计计算机系统时，更好地理解软硬件之间的关系，并且更好地利用它们来实现指定的逻辑功能。例如，如果我们知道某个逻辑功能可以用软件实现，那么我们就可以避免使用更复杂、更昂贵的硬件来实现这个功能。

什么是固件？举例说明固件在计算机系统中的应用。

固件是一种用于控制计算机硬件设备的软件。它通常存储在可编程存储器中，例如 ROM、EPROM 或 Flash，并且与硬件设备相关联。固件不同于普通的软件，因为它不能被用户直接更改或修改，而且在系统启动时就会被自动加载。

一个典型的例子是计算机的 BIOS（基本输入/输出系统）。这是一个固件程序，它在计算机启动时自动加载，并负责执行一些基本的初始化任务，例如检查系统的硬件配置、识别硬盘驱动器和其他设备、加载操作系统等。

另一个例子是网络设备，例如路由器或交换机。这些设备通常都包含一个固件程序，用于控制设备的硬件功能，并执行网络协议和其他相关任务。固件在这些设备中发挥着重要作用，可以确保设备能够正常工作，并实现指定的网络功能。

浮点数

浮点数 \(N=M\times R^E\) 。

尾数 \(M\) 为定点小数；阶码 \(E\) 为定点整数；阶码的底 \(R\) 一般为2、8或16，隐含规定。

规格化表示

充分利用尾数的二进制数位来表示更多的有效数字，将尾数的绝对值限定在某个范围之内。

尾数 \(M\) 应满足条件：绝对值最高有效位为1；要留意尾数的编码方式

定点数的乘法

定点数的乘法主要从采用原码和补码的方式实现。我这里这讨论原码的方式。

原码乘法步骤

这里说的是 原码一位乘法

初始化：\(X\cdot Y\) ，部分积为0

  1. 符号位单独处理，数据的绝对值参与乘法运算
  2. 若 \(Y_i=1\) ，部分积加 \(|X|\) ；若 \(Y_i=0\) ，部分积加 \(0\) 。累加结果右移一位，产生新的的部分积。
  3. 累计右移 \(n\) 次。

PS：部分积采用保留一位附加符号位的方式，用于暂存累加时出现的溢出位，后续右移时可直接移入数值位。

阵列乘法器

下图是 4位乘4位的绝对值阵列乘法器。其中 FA 是全加器。

虽然我们没有讲补码的乘法运算。但只要在绝对值阵列乘法器的基础上，添加一个异或门产生积的符号，再让原码数据通过一个 补码求绝对值的电路 把补码转成绝对值送入绝对值阵列乘法器，算出积的绝对值，再通过一个 绝对值求补码的电路 把绝对值转回补码形式，最终就得出了积的补码形式。

其实这两个电路实现是一样的。

层次结构

DRAM的刷新

现在的微型计算机中大都采用 DRAM 作为其主存。

DRAM 采用读出的方式进行刷新。所有的 DRAM 都需要刷新操作。

刷新方式 描述
集中式刷新 在一个刷新周期中，用一段时间对DRAM的所有行进行刷新，期间禁止读写操作，这段时间称为访存死区
分散式刷新 把对每一行的数据刷新分散到各个存储周期去
异步式刷新 把对每一行的数据刷新直接分散

主存储器与CPU的连接

现代微型计算机（PC）的主存储器基本包括两大部分。

一个是ROM，它存放的是不受电源影响的非易失性存储器，是生产厂家烧写的固定指令和数据。

一个是RAM，断电数据会丢失，但主存的绝大部分都是由RAM构成，RAM对主存性能起着决定性作用。

高速缓冲存储器Cache

主存速度的提高始终跟不上CPU的发展。据统计，CPU的速度平均每年提高60%，而组成主存的DRAM的速度平均每年只改进7%。由SRAM组成的高速缓冲存储器的运行速度则接近甚至等于CPU的速度。

提示速度：在CPU和主存之间增加小容量快速的SRAM→Cache。

理论依据：

几点说明：

  • 在相对容量较大而速度较慢的主存与高速处理器之间设置少量但快速的存储器
  • CPU的访存操作都是首先访问Cache
  • 为追求高速，包括管理在内的全部功能由硬件实现
  • Cache的内容是主存部分内容的副本，Cache的功能均由硬件实现，对程序员是透明的

几个概念：

  • 高速命中(Hit)：在Cache中找到
  • 高速失效(Miss)、缺失、未命中：在Cache中没找到

地址映射方式

Cache 的数据块大小称为行，主存的数据块大小称为块，两者等长。

利用某种方法或者规则将主存块定位到cache，称为地址映射。

现设 \(i\) 表示主存的块号，\(j\) 表示Cache的行号，主存有 \(2^m\) 块，Cache 有 \(2^c\) 块。

Cache有 行地址、标记区、行数据 。

直接映射

一种多对一的映射方式。主存与Cache之间数据块的对于关系： \(j=i\ mod\ 2^c\) 。

这个映射方式中，一个主存地址码可以这样划分：高位标记、字块地址、块内地址。

区 描述
字块地址 \(i\) 用来定位到Cache的某行（每个地址码都会定位到Cache的某行
高位标记 用来确定当前访问的地址其所在数据块是否在Cache中（若与Cache存的标记相等说明命中
块内地址 用来定位数据块中的某一数据
全相联映射

全相联映像就是让主存中任何一个块均可以映像装入到Cache中任何一个块的位置上。

这个映射方式中，一个主存地址码可以这样划分：高位标记、块内地址。

区 描述
高位标记 用来确定当前访问的地址其所在数据块是否在Cache中（若与Cache存的标记相等说明命中
块内地址 用来定位数据块中的某一数据
组相联映射

组相联映射是前两种方法的折中方案。它把 Cache分组，然后组间采取直接映像，而组内采取全相联映像。

设分成 \(2^{c-r}\) 组，每组就有 \(2^r\) 行，有如下关系：\(j=(i\ mod\ 2^{c-r})\times 2^r+k\) ，其中 \(+k\) 就是组内偏移。

这个映射方式中，一个主存地址码可以这样划分：高位标记、组地址、块内地址。

区 描述
组地址 用来定位到Cache的某组（每个地址码都会定位到Cache的某组
高位标记 用来定位到Cache组内的某行（若与Cache存的标记相等说明命中
块内地址 用来定位数据块中的某一数据

替换策略

CPU访问Cache未命中的情况下，会将包括未命中的存储单元所在的整个数据块调入Cache。如果Cache的存储空间已满，则需要替换现有某个数据块。新主存块要进入Cache，决定替换哪个原主存块的方法，称为“替换策略”。

直接映射，因为是唯一映射，替换唯一的一个Cache行即可。

全相联和组相联比较复杂，有以下一些替换算法：

  • 随机替换算法
  • 先进先出算法
  • 最近最少使用算法

算法的具体顾名思义，这里不细讲。

写策略

CPU写入操作命中Cache的情况下，可能会导致Cache的内容与主存的内容不一致，为避免Cache与主存内容不一致的问题所采用的方法，称为“写策略”。

常用的有两种：

  • 写贯穿策略
    • CPU写Cache命中：同时写Cache和写主存
    • CPU写Cache不命中：直接写主存，有两种做法：写入和不写入Cache
  • 写回策略
    • 无论CPU写Cache命中还是不命中：只写Cache，不修改主存，仅当此Cache行被换出时，相应的主存内容才被修改

控制器

我们知道，控制器一般和运算器制作在同一芯片上——中央处理器是控制计算机自动完成取出指令和执行指令任务的部件。它是计算机的核心部件，通常简称为CPU（Central Processing Unit）

CPU组成和功能

CPU具有以下4个基本功能：

功能 描述
指令控制 保证机器按程序规定的顺序取出执行
操作控制 CPU产生每条指令所对应的操作信号，并把各种操作信号送往相应的部件，从而控制这些部件按指令的要求进行动作
时间控制 对各种操作的执行严格按照时间序列进行
数据加工 对数据进行算术运算和逻辑运算处理（运算器的功能

对于一个CPU来说，它有6个专用寄存器：

寄存器 描述
数据缓冲寄存器（DR） 暂时存放由内存读出或写入的指令或数据字
指令寄存器（IR） 保存当前正在执行的一条指令
程序计数器（PC） 确定下一条指令的地址

控制器是由 专用寄存器、指令译码器、操作控制器、时序信号产生器 组成。

部件 描述

操作控制器 根据指令的操作码以及时序信号，产生取出指令和执行这条指令所需的各种操作控制信号

时序控制

先来看看计算机中与时序有关的几个概念：

周期 描述
指令周期 是指计算机从内存取出一条指令并完成该指令的执行所需要的时间。
机器周期（CPU周期 一般指CPU与内存交换一次信息（读或写）所需要的时间。
时钟周期（节拍周期 是指CPU执行一个微操作命令（即控制信号）的最小时间单位，即T周期。

时序信号发生器

一般都是由 时钟源、环形脉冲发生器、节拍脉冲逻辑、启停控制逻辑 四个部分组成。

控制方式

一条指令有几个机器周期？每个机器周期又有几个节拍？

控制方式 描述
同步控制方式 以微操作序列最长的指令为标准，确定控制微操作运行的时钟周期数（节拍数）
异步控制方式 每条指令需要多少节拍，就产生多少节拍
联合控制方式 把同步控制方式和异步控制方式结合使用的一种方式

联合控制方式是这样的：大部分指令安排在统一的机器周期内完成，即同步控制；而将较少数特殊指令，或微操作序列过长或过短，或微操作时间难以确定的，采用异步控制来完成。

现代计算机系统大多采用联合控制方式，其一般设计思想是，在功能部件内部采用同步控制方式，而在功能部件之间采用异步控制方式。
